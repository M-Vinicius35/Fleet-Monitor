## 🇺🇸 English Version

This project is a robust fleet monitoring solution, developed with a focus on **scalability**, **performance**, and **data reliability**. The application was designed following **Clean Architecture** and **Clean Code** principles, ensuring the system is highly maintainable and testable.

### 🎯 Objective
Fleet managers often struggle to visualize vehicle positions and statuses in real-time. **Fleet Monitor** centralizes this data into an intelligent dashboard, providing:
* Precise telemetry visualization.
* Reduced latency in data updates.
* Intuitive interface for rapid alert response.

### 🛠️ Tech Stack
* **Frontend:** React.js + Vite + TypeScript.
* **Backend:** Node.js + TypeScript.
* **Communication:** Socket.io (Real-time data streaming).
* **Styling:** Tailwind CSS.

### 🧠 Engineering & QA Mindset
Unlike generic projects, this was built with a QA Engineer's mindset:
* **Separation of Concerns (SoC):** Business logic is strictly isolated from the UI.
* **Clean Code:** Small, modular functions that facilitate **Unit Testing**.
* **QA Strategy:** Structured to support automation using tools like **Cypress** or **Playwright**.

### ⚙️ How to Run
1. **Backend:** `cd backend && npm install && npm run dev`
2. **Frontend:** `cd frontend && npm install && npm run dev`

## 🇧🇷 Versão em Português

Este projeto é uma solução robusta para o monitoramento de frotas, desenvolvida com foco em **escalabilidade**, **performance** e **confiabilidade de dados**. A aplicação foi projetada seguindo os princípios de **Clean Architecture** e **Clean Code**.

### 🎯 Objetivo
Gestores de frota muitas vezes enfrentam dificuldades em visualizar a posição e o status dos veículos em tempo real. O **Fleet Monitor** centraliza esses dados em um dashboard inteligente:
* Visualização precisa de telemetria.
* Redução de latência na atualização de dados.
* Interface intuitiva para resposta rápida a alertas.

### 🛠️ Stack Tecnológica
* **Frontend:** React.js + Vite + TypeScript.
* **Backend:** Node.js + TypeScript.
* **Comunicação:** Socket.io (Fluxo de dados em tempo real).
* **Estilização:** Tailwind CSS.

### 🧠 Mindset de Engenharia e QA
Este projeto foi construído com a mentalidade de um Engenheiro de QA:
* **Separação de Responsabilidades (SoC):** Lógica de negócio isolada da interface.
* **Clean Code:** Funções pequenas que facilitam a criação de **Testes Unitários**.
* **Estratégia de QA:** Estruturado para suportar automação com **Cypress** ou **Playwright**, focando no "Happy Path" e tratamento de erros.

### ⚙️ Como Executar
1. **Backend:** `cd backend && npm install && npm run dev`
2. **Frontend:** `cd frontend && npm install && npm run dev`
